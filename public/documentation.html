API Endpoints

POST /api/v1/identify

Core endpoint for contact identification and reconciliation. Processes contact information and returns consolidated customer data.

Request Body:
{
  "email": "customer@fluxkart.com",
  "phoneNumber": "1234567890"
}
Response Format:
{
  "contact": {
    "primaryContatctId": 1,
    "emails": ["customer@fluxkart.com", "alt@email.com"],
    "phoneNumbers": ["1234567890", "0987654321"],
    "secondaryContactIds": [2, 3]
  }
}

GET /api/v1/health

Health check endpoint that returns service status and database connectivity.

Response:
{
  "status": "healthy",
  "service": "FluxKart Identity Service",
  "timestamp": "2024-05-28T12:00:00.000Z",
  "uptime": 3600,
  "database": "connected",
  "version": "1.0.0"
}

GET /api/v1/status

Detailed system status including server information and performance metrics.

Usage Examples

1. New Customer

First-time customer registration creates a primary contact.

curl -X POST /api/v1/identify \
  -H "Content-Type: application/json" \
  -d '{"email": "new@customer.com", "phoneNumber": "555-0001"}'
Creates primary contact with ID 1
2. Contact Linking

Same customer uses different email with same phone number.

curl -X POST /api/v1/identify \
  -H "Content-Type: application/json" \
  -d '{"email": "work@customer.com", "phoneNumber": "555-0001"}'
Links to existing primary via phone match
3. Contact Merging

Request bridges two separate primary contacts.

curl -X POST /api/v1/identify \
  -H "Content-Type: application/json" \
  -d '{"email": "customer1@email.com", "phoneNumber": "555-0002"}'
Merges two contact trees intelligently
